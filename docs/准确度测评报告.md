# 准确度测评报告

## 1. 任务定义

判断“任命证书”文本是否合理：
- 正例（label=1）：表达与逻辑合理
- 反例（label=0）：存在明显逻辑问题

指标：Accuracy、Precision、Recall、F1（正类=合理）。

## 2. 数据集

- 文件：`当前目录/data/appointment_cert_dataset.jsonl`
- 规模：200 条（100 正例 + 100 反例）
- 生成脚本：`当前目录/tests/gen_appointment_dataset.py`

重生成：

```bat
cd /d d:\路径\当前目录
python tests\gen_appointment_dataset.py
```

## 3. 评测脚本

- 脚本：`当前目录/tests/accuracy_eval.py`
- 输出：
  - `accuracy_full.json`（明细）
  - `accuracy_summary.txt`（摘要）

单独运行示例：

```bat
cd /d d:\路径\当前目录
python tests\accuracy_eval.py --dataset data\appointment_cert_dataset.jsonl --out-json test_results\accuracy_full.json --out-txt test_results\accuracy_summary.txt
```

## 4. 一键流程参数（推荐）

通过 `run_all.bat` 控制准确度速度与稳定性：

- `ACCURACY_LIMIT=20`：快速试跑
- `ACC_N_PREDICT=16`（或 32）：避免答案过早截断
- `ACC_NGL=0`（GPU 可试 999）

示例：

```bat
set ACCURACY_LIMIT=20
set ACC_N_PREDICT=32
set ACC_NGL=0
run_all.bat
```

## 5. 常见异常说明

- **Accuracy≈0.5 且 P/R/F1≈0**：常见于模型几乎不输出可解析答案，或过度偏向单一类别。
- **`pred=null` 较多**：输出格式不稳定；可提高 `ACC_N_PREDICT`，并查看 `accuracy_full.json` 的 `tail` 字段排查。
- 默认小模型（SmolLM2 135M）中文判别能力有限，建议在正式报告中说明模型能力边界。
