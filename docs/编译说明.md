# 编译说明

## 1. 目录定位

本作业以 **当前目录为独立交付单元**。在其他电脑拉取后，只需进入在本地文件夹运行。

文件夹中关键内容：

- `run_all.bat`：一键完整流程（编译 + 性能 + 准确度）
- `CMakeLists.txt`：构建 `local_llm.exe`
- `tests/`：评测脚本
- `data/appointment_cert_dataset.jsonl`：200 条数据（100 正例 + 100 反例）
- `docs/`：作业文档

## 2. 环境要求

- Windows 10/11 x64
- Git（在 PATH）
- CMake 3.14+（在 PATH）
- Python 3.9+（在 PATH）
- Visual Studio 2022（含 C++ 工具链，推荐）

## 3. 一键运行

在当前目录双击：

- `run_all.bat`

脚本会自动：
1) 克隆/更新 `当前目录/llama.cpp`
2) 编译并生成 `local_llm.exe`
3) 下载 GGUF 到 `当前目录/models/`
4) 执行 smoke + bench + perf + accuracy
5) 生成结果到 `当前目录/test_results/run_时间戳/`

## 4. 手动构建（可选）

```bat
cd /d d:\路径\当前路径
cmake -S . -B build
cmake --build build --config Release -t local_llm_exe
```

常见可执行路径：
- `当前路径\build\bin\Release\local_llm.exe`
- 或 `当前路径\llama.cpp\build\bin\Release\llama-completion.exe`（脚本会自动兼容）

## 5. 运行参数（run_all.bat）

- `SKIP_BUILD=1`：跳过编译
- `SKIP_PERF_PY=1`：跳过 `perf_smoke_test.py`
- `SKIP_PIP_MPL=1`：不自动安装 matplotlib
- `SKIP_ACCURACY=1`：跳过准确度
- `ACCURACY_LIMIT=20`：只跑前 N 条
- `ACC_N_PREDICT=16`：准确度单条生成 token 数（默认 16）
- `ACC_NGL=0`：准确度评测 ngl（GPU 可设 999）
- `SKIP_ROOT_LAUNCHER=1`：不编译 `当前路径/local_llm.exe` 启动器

示例（快速调试）：

```bat
set ACCURACY_LIMIT=20
set ACC_N_PREDICT=32
set ACC_NGL=0
run_all.bat
```

## 6. 常见问题

- **CMake 报 `nmake` / `C_COMPILER not set`**：未安装或未加载 VS C++ 工具链；安装 VS Build Tools 并重试。
- **准确率 0.5、P/R/F1 接近 0**：模型未稳定输出目标格式或中文能力弱；提高 `ACC_N_PREDICT`，必要时换中文能力更强模型。
- **模型下载失败**：检查网络或手动把 GGUF 放到 `f/models/`。
