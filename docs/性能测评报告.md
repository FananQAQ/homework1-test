# 性能测评报告（模板 + 说明）

## 1. 测评目标（对应 need.md）

| 指标 | 含义 | 本仓库实现方式 |
|------|------|----------------|
| **TTFT** | 从输入完成到**首个输出 token** 的时间 | `llama-completion` / `local_llm` 非流式输出，采用 **估计值**：`prompt_eval_ms + (eval_ms / gen_runs)`，其中 `prompt_eval` 对应 prefill，`eval_ms/gen_runs` 近似单步 decode。精确 TTFT 需 `llama-server` 流式接口或自写 C API 打点。 |
| **TPOT** | 每个输出 token 平均耗时 | 由 `--perf` 输出中的 `eval time ... / N runs` 得 **TPOT ≈ eval_ms / N**。 |
| **E2E** | 端到端完成本次生成的总时间 | **子进程墙钟时间**（含加载模型、prefill、生成固定 `n` 个 token）。 |

## 2. 测试方法

- 可执行文件：`local_llm.exe`（由根目录 `CMakeLists.txt` 自 `llama-completion` 复制得到）。
- 脚本：`tests/perf_smoke_test.py`  
  - **≥12 组**不同字符长度的输入（满足「至少 10 组」）。  
  - 固定 `-n`（默认 32）、`-ngl 0`（CPU），`--perf` 打开性能打印。  
  - 输出：`test_results/perf_smoke_<UTC时间>.json`、`.md`，若安装 `matplotlib` 则额外生成 **`.png` 图表**（三张柱状子图：E2E、TTFT 估计、TPOT）。

运行示例：

```bat
pip install -r requirements.txt
python tests\perf_smoke_test.py
```

一键流程（含编译与下载模型）：

```bat
run_all.bat
```

## 3. 数据与图表（填写区）

将最近一次运行的文件路径与关键结论填在此处：

- JSON 数据：`test_results/perf_smoke_*.json`
- Markdown 表：`test_results/perf_smoke_*.md`
- 图表 PNG：`test_results/perf_smoke_*.png`

**结论摘要（请根据你的机器填写）：**

- 硬件：CPU 型号 / 内存 / 是否 GPU  offload（本脚本默认 CPU）。
- 模型：GGUF 文件名与量化类型。
- 观察：随 prompt 长度增长，TTFT 估计与 E2E 的变化趋势；TPOT 是否相对稳定。
