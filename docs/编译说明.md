# 编译说明（llama.cpp → local_llm.exe）

## 1. 环境与依赖

- Windows 10/11 x64（本文以 Windows 为主；Linux/macOS 可将 `local_llm.exe` 换为 `local_llm`）。
- [CMake](https://cmake.org/) 3.14+，已加入 `PATH`。
- [Git](https://git-scm.com/)，已加入 `PATH`。
- **Visual Studio 2022**（含「使用 C++ 的桌面开发」工作负载），或 **「x64 Native Tools Command Prompt for VS」** 可用的 MSVC 工具链。

可选：若希望 `llama-server` 等支持 HTTPS 模型下载，可安装 OpenSSL 并配置 `OPENSSL_ROOT_DIR`（当前作业脚本使用直链下载 GGUF，可不装）。

## 2. 获取源码

在作业仓库根目录（与 `CMakeLists.txt` 同级）执行：

```bat
git clone --depth 1 https://github.com/ggerganov/llama.cpp.git llama.cpp
```

若已克隆，保持 `llama.cpp` 子目录即可。

## 3. 使用本仓库 CMakeLists.txt 构建

本仓库根目录的 `CMakeLists.txt` 会 `add_subdirectory(llama.cpp)`，并打开 `LLAMA_BUILD_COMMON` / `LLAMA_BUILD_TOOLS`，关闭示例与 server，以加快构建；构建完成后将 `llama-completion` **复制为 `local_llm.exe`**（满足 `need.md` 对可执行文件命名的要求）。

在仓库根目录执行：

```bat
cmake -S . -B build
cmake --build build --config Release -t local_llm_exe
```

生成物路径（多配置生成器）一般为：

```text
build\bin\Release\local_llm.exe
```

**重要（Windows）——两种 `local_llm.exe`：**

1. **`build\bin\Release\local_llm.exe`（或 `llama.cpp\build\...\local_llm.exe`）**：真正的推理程序，须与同目录下的 `ggml.dll`、`llama.dll` 等在一起。在 **CMD** 中带参数调用，例如：

```bat
"d:\路径\hmwk\build\bin\Release\local_llm.exe" -m "d:\路径\hmwk\models\SmolLM2-135M-Instruct-Q4_0.gguf" -no-cnv -p "你好" -n 64 --perf
```

2. **仓库根目录的 `local_llm.exe`**：`run_all.bat` 成功结束后会用 MSVC 编译 `launcher\main.c` 生成的小型**启动器**（若本机找不到 VS C++ 工具链会跳过并给出提示）。**无参数双击**等价于再跑一遍 `run_all.bat`（完整测试流程）；**带参数**时会把参数转发给上面第 1 条路径下的推理 `local_llm.exe`。不要在根目录手动创建 **0 字节** 的 `local_llm.exe` 占位；`run_all.bat` 会删除 0 字节假文件。

对话模式（若模型带 chat template）可参考上游文档使用 `-cnv` / `--jinja` 等参数。

## 4. 常见问题

| 现象 | 处理 |
|------|------|
| `Missing llama.cpp/CMakeLists.txt` | 未克隆子目录，按第 2 步执行 `git clone`。 |
| CMake 找不到 C++ 编译器 | 打开「x64 Native Tools Command Prompt for VS」再执行 cmake，或安装 VS C++ 工作负载。 |
| 仅存在 `llama-completion.exe` 而无 `local_llm.exe` | 未构建目标 `local_llm_exe`，或使用了仅针对 `llama.cpp` 子目录的旧 `build` 目录；请用**仓库根**的 `cmake -S . -B build` 重新配置并构建 `local_llm_exe`。 |
| OpenSSL 警告 | 可忽略（不影响 CPU 推理）；需要 HF HTTPS 时再装 OpenSSL。 |

## 5. 一键脚本

根目录 **`run_all.bat`**：拉取/更新 `llama.cpp`、编译、`local_llm.exe`、下载 GGUF；每次运行会在 **`test_results\run_年月日_时分秒\`** 下生成**当次**全部输出：

- `test_report.txt`（冒烟 + bench + 附录）
- `perf_smoke_*.json` / `.md` / `.png`
- `accuracy_full.json`、`accuracy_summary.txt`（准确率、精确率、召回率、F1；基于 `data\appointment_cert_dataset.jsonl`）

Python 测试脚本位于 **`tests\`**。在本包根目录双击 **`run_all.bat`** 即可完整编译与测试（与上级目录中的 `hmwk\run_all.bat` 效果相同，后者会转发到本目录）。

环境变量（可选）：

- **`SKIP_BUILD=1`**：跳过编译  
- **`SKIP_PERF_PY=1`**：跳过性能多组脚本  
- **`SKIP_PIP_MPL=1`**：不自动 `pip install matplotlib`  
- **`SKIP_ACCURACY=1`**：跳过准确度评测（200 条很慢）  
- **`ACCURACY_LIMIT=20`**：只评前 N 条（调试/快速看指标）
- **`SKIP_ROOT_LAUNCHER=1`**：不在末尾编译仓库根目录的启动器 `local_llm.exe`

双击运行结束后会 **`pause`**。
