# 准确度测评报告 — 「任命证书」是否合理

## 1. 任务说明（need.md）

- **正例**：中文表达与常识逻辑上**合理**的任命类证书（本仓库用程序生成 **100** 条）。
- **反例**：存在明显矛盾、荒诞日期、自我循环任命等 **不合理** 样本（**100** 条）。
- **模型任务**：对每条文书判断「合理 / 不合理」。
- **指标**：整体准确率；将「合理」视为正类时的 **精确率、召回率、F1**。

## 2. 数据集

- 生成脚本：`tests/gen_appointment_dataset.py`
- 数据文件：`data/appointment_cert_dataset.jsonl`（每行 JSON：`id`, `label`（1=合理，0=不合理）, `text`）

重新生成：

```bat
python tests\gen_appointment_dataset.py
```

## 3. 评测脚本

- 脚本：`tests/accuracy_eval.py`
- 默认自动查找 `local_llm.exe` 或 `llama-completion.exe`（`build\bin\Release` 或 `llama.cpp\build\bin\Release`）。
- 每条样本一次子进程调用，**200 条全量评测耗时较长**，调试可用 `--limit 20`。

示例：

```bat
python tests\gen_appointment_dataset.py
python tests\accuracy_eval.py --out-json test_results\accuracy_full.json
```

控制台会打印 JSON 字段 `accuracy`、`precision_positive` 等。`run_all.bat` 生成的 **`accuracy_summary.txt` 为中文说明**（准确率、精确率、召回率、F1、混淆矩阵 TP/FP/TN/FN）。
