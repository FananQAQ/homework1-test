# 技术总结

## 1. 交付结构

本作业以当前文件夹为独立交付目录，包含：
- 构建：`CMakeLists.txt`、`run_all.bat`
- 测试：`tests/`
- 数据：`data/appointment_cert_dataset.jsonl`
- 文档：`docs/`

这样在任意机器拉取后，进入 `当前目录/` 即可一键执行完整流程。

## 2. 关键实现

- 使用 `llama.cpp` 源码构建，并将 `llama-completion` 交付为 `local_llm.exe`
- 性能脚本覆盖 >=10 组不同输入长度，输出 JSON/MD/PNG
- 准确度脚本对 200 条数据计算 Accuracy/Precision/Recall/F1
- `run_all.bat` 支持环境变量控制（限量、跳过模块、准确度参数）

## 3. 主要问题与处理

1) **跨机器构建失败（nmake/cl 缺失）**
- 原因：目标机器缺少 VS C++ 工具链
- 处理：安装 VS Build Tools，并在脚本中增强工具链探测

2) **准确度指标异常（P/R/F1 为 0）**
- 原因：模型输出格式不稳定、可解析答案少
- 处理：调整解析逻辑、提高 `ACC_N_PREDICT`，并保留 `accuracy_full.json` 便于回溯

3) **200 条评测耗时偏长**
- 处理：提供 `ACCURACY_LIMIT` 快速模式，支持 GPU 参数 `ACC_NGL`

## 4. 经验与后续优化

- 小模型在中文判别任务上的稳定性有限，报告需注明模型边界
- 可改为服务常驻（llama-server）减少每条子进程启动开销
- 可引入更强中文模型或输出约束（结构化标签）提升可解析率
